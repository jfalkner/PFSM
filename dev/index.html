Peptide Finite State Machine Project (Version 1.0, "The Paper was Published")

This project provides a free, open-source, Java implementation of a Peptide Finite State Machine (see Falkner, Andrews, Fast tandem mass spectra-based protein identification regardless of the number of spectra or potential modifications examined, Bioinformatics, 2005). The code may be used to quickly scan through protein sequences, e.g. any FASTA file, and matches subsequences to a peak list. PFSM are well suited for this task as they allow any number of potential modifications to be considered or any number of peak lists to be multiplexed without resulting in a longer search time.

What the code does may seem like magic, but the technique used will be familiar to any computer scientist. If you convert mass spectra in to a single regular expression you can side-step almost all the bottle-necks of common mass spec search algorithms. It is slick, check out the code.

Features

PFSMs may be used to quickly identify potential peptide sequences from a set of protein sequences very quickly, regardless of the number of peak lists you are searching or the number of potential modifications considered. This is the key feature of PFSMs and it is described well in the algorithm's paper. Other helpful features are summarized here.

Free and Open-Source
This project is free to use, as stated in the license, and it is completely open-source. The goal is to give proteomics researches a good, open, free tool to use for MSMS-based protein identification.
Cross Platform Compatability, Dynamic Code Updates
All code in this project is implemented in the Java programming language, and it may be run on any platform supporting Java, including Linux, MS Windows, OSX, BSD, and Solaris. This project also uses Java Web Start to allow for dynamic web-based updates. You'll always automatically have free access to the latest version of the code.
Database File Formats
FASTA files may be used as input.
Peak List/Spectra Formats
This project is based on the ProteomeCommons.org-IO framework, and it supports many different MS and MSMS data file formats such as MGF, DTA, and PKL.

Requirements

You will need a copy of the Java Runtime Environment (JRE) 1.4+ in order to run this project. A free copy of the JRE may be obtained at java.sun.com. There is no formal requirement on processor speed or RAM, but more of either will generally make things go faster.

Running the code

In this beta release of the code is a utility that will construct a PFSM from a set of peak lists and use it to screen a FASTA file. An example is provided and it may be run by using the following command.

java -jar Falknew-PFSM.jar default-config.xml example.fasta peaklist-1.mgf peaklist-2.mgf peaklist-3.mgf peaklist-4.mgf

The program will examine the FASTA file example.fasta using the PFSM to pick out which fragments of a protein could satisfy the given peaklist. Results are displayed on the screen, sorted by showing proteins with the most peptides matched first.

In general the code may be run by using the following command.

java -jar Falkner-PFSM.jar <config file> <fasta file> <peak list file(s)>

Where you replace the config file with your configuration (default-config.xml should be fine for TOFTOF data), the FASTA file with any FASTA file you please, and the peak lists files with any peak list files you may have.
